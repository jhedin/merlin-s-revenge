# Constant-Drift Audit — 2026-06-24 03:52:23Z

**Lens (AUDIT-CHARTER):** a wrong gameplay constant still RUNS — no error, no failing test — it just makes
the game feel/behave subtly off. The blind spot is a magic number HARDCODED in the port that should equal a
cast value (or should be read per-actor from `data.json`). The class already bit us once (`atCapacity()`
hardcoded 12/16 where the cast gates on each team's `#maxMembers`). This sweep grepped the port src for
hardcoded gameplay numerics (cooldowns, speeds, scales, knock magnitudes, thresholds, frame durations,
counts/caps, friction, ranges, increments, probabilities, `?? N` fallbacks), then cross-checked each against
`casts/` and `port/src/generated/data.json`.

Method: static grep + cast lookup, with a registry probe (`tools/_audit_const_speed.ts`, gitignored,
removed after use) confirming `registry.resolveActor("merlin").walkSpeed === 4` (1:1 with the cast, not
px-scaled) — which is what makes the speed-potion increment a true drift.

Beware-calibration items (KNOCK_*, MELEE_SCALE, ENEMY_DAMAGE_SCALE, BULLET_DAMAGE_SCALE, SPELL_FX,
`dmgRef`) are **documented deliberate px-slice re-tunings** (K1/K2/B2 plans) and are marked FAITHFUL — not
"fixed."

## Bottom line

One clear, fixable drift (**speed potion +0.6 vs cast +0.075 — 8×**), one trusted-divergence suspect worth
re-deriving (**nav-mode ×3 vs an apparent ~8× cap-bound original**), and one documented simplification

constants are all faithful to the cast; the damage/knockback scales are documented px-slice calibrations and
were left untouched.
