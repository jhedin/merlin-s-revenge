# Merlin's Revenge — TypeScript/HTML5 port

A from-scratch reimplementation of the Director/Lingo engine (see `../docs/PORTING_PLAN.md`
and `../docs/PLAN_REVIEW.md`). Game data and assets come from `../casts`, `../maps`, and
`../extracted` (regenerated by the tools below — not committed).

## Status

Playable through **Phases 0–7** of the roadmap: title screen → a real 10-room dungeon with tile
collision and per-room spawn tables, a player + AI enemies (chase, melee & ranged, HP/death),
projectiles with object pooling, XP/leveling, save/load, and a game-over/restart flow — all at a
fixed 30 Hz through the component dispatch. Controls: **WASD/arrows** move, **space** attack,
**1/2** save/load.

Still unported (Phases 6 cutscenes / 8 content): the cutscene engine, construction/army economy,
status effects, summoning, full menus, and the remaining ~115 actor types' sprites/behaviors.

## Run

```sh
cd port
npm install
npm run parse-data     # ../casts/data/*.txt -> src/generated/data.json
npm run build-assets   # ../extracted + tilesets/sprites -> public/assets + src/generated/assets.json
npm run dev            # vite dev server
```

`npm test` runs the unit suite; `npm run typecheck` runs `tsc`.

## Architecture (implemented so far)

- **`src/engine`** — the kernel. `dispatch.ts` is the heart: Entity/Component/Archetype with the
  four-primitive continuation dispatch (forward / shadow / fold / `me.big` full re-entry) that
  faithfully models Lingo's ancestor chain (`PLAN_REVIEW.md` §1). Handler refs are resolved at
  archetype-build time (no per-call megamorphic lookup). Plus `events`, `updater`, fixed-step
  `loop`, and `math` (alloc-conscious vectors + deterministic RNG).
- **`src/data`** — `lingo.ts` parses the Lingo literal grammar (all 295 real records);
  `registry.ts` does `#inherit` flattening + `#attack` schema merge (resolves 263/263 actors);
  `tlk.ts` parses tile keys.
- **`src/world`** — `map.ts` (per-map room sizes), `collision.ts` (tile AABB resolver).
- **`src/render`** — Canvas2D z-sorted display list with registration-point alignment; matte
  transparency keyed at load (no per-frame `getImageData`).
- **`src/components` / `src/entities`** — `Movement` (objMoveXY), `PlayerControl`/`EnemyAI`,
  `Anim` (modAnimSet), `Energy`/`Team`, composed into player/enemy archetypes.

## Tooling (`tools/`)

- `parse_data.ts` — build-time Lingo data → JSON (+ validation/stats).
- `build_assets.ts` — copies tilesets + character frames (with regpoints) into `public/assets`.
- `screenshot.ts` — headless-chromium verification harness (drives input, screenshots, reads state).

## Not yet ported

Projectiles & ranged/magic weapons, leveling/XP, status effects, multi-room navigation & screen
exits, construction/army economy, cutscenes/menus/scene-flow, and save/load. See the phased
roadmap in `../docs/PORTING_PLAN.md`.
